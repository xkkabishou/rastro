# PRD — Obsidian 笔记同步功能 (v4)

> Genesis v4 | 2026-03-20

## 1. 背景与目标

### 1.1 问题描述
用户在 Rastro 中阅读论文时，会生成 AI 总结和聊天记录。目前这些内容仅存储在应用内部数据库中，无法与用户的知识管理系统（Obsidian）联动。用户希望将这些内容以 Markdown 格式导出到 Obsidian Vault，以便在更广泛的知识体系中管理和检索。

### 1.2 核心目标
- **手动导出**：用户可一键将总结导出到 Obsidian
- **选择性导出聊天记录**：用户可多选聊天会话批量导出
- **自动同步**：总结生成完成后自动写入 Obsidian（可配置开关）
- **结构化文件夹**：在 Obsidian 中以 `文献笔记/{文献名}/` 组织文件
- **可扩展**：预留多种总结类型（详细、简略等）的扩展空间

## 2. 用户故事

### US-1: 配置 Obsidian Vault 路径
**[REQ-401]** 作为用户，我想在设置中配置我的 Obsidian Vault 路径，以便应用知道将文件写入哪里。

**验收标准**：
- Given 用户打开设置面板
- When 用户在"Obsidian 同步"配置项中选择或输入 Vault 路径
- Then 路径被保存到应用配置中
- And 路径有效性被验证（目录存在 + 可写）

### US-2: 手动导出总结到 Obsidian
**[REQ-402]** 作为用户，我想将当前文献的 AI 总结一键导出到 Obsidian。

**验收标准**：
- Given 当前文献已生成 AI 总结 且 Obsidian Vault 已配置
- When 用户点击"导出到 Obsidian"按钮
- Then 总结被写入 `{Vault}/文献笔记/{文献名}/总结.md`
- And 文件包含 YAML front matter（标题、日期、来源）
- And 导出成功后显示确认反馈

### US-3: 选择性导出聊天记录
**[REQ-403]** 作为用户，我想选择性地将某些聊天记录导出到 Obsidian。

**验收标准**：
- Given 当前文献有聊天记录 且 Obsidian Vault 已配置
- When 用户打开导出面板，看到聊天会话列表
- Then 用户可通过 checkbox 多选要导出的会话
- And 点击导出后，每个选中的会话导出为独立 `.md` 文件
- And 文件命名格式为 `对话-{日期时间}.md`

### US-4: 自动同步
**[REQ-404]** 作为用户，我想让总结生成完成后自动同步到 Obsidian。

**验收标准**：
- Given 用户在设置中启用了"自动同步"开关 且 Vault 路径有效
- When AI 总结生成完成（`finishStream` 触发）
- Then 总结自动写入 Obsidian 对应的文献文件夹
- And 如果已有同名文件，自动覆盖更新
- And 如果 Vault 路径失效（目录不存在），在控制台记录警告但不中断用户流程

### US-5: 多种总结类型（扩展预留）
**[REQ-405]** 作为用户，未来我想生成不同类型的总结（详细、简略），每种类型对应独立的 md 文件。

**验收标准**：
- Given 文献文件夹 `{Vault}/文献笔记/{文献名}/` 
- When 存在多种总结类型时
- Then 各类型总结以不同文件名存储（如 `总结-详细.md`、`总结-简略.md`）
- And 现有"默认总结"使用 `总结.md` 文件名

## 3. 功能需求

### 3.1 设置面板 — Obsidian 配置区块

| ID | 需求 | 优先级 |
|---|---|---|
| REQ-401a | 设置面板新增 Obsidian 配置卡片（在存储管理 tab 或独立 tab 中） | P0 |
| REQ-401b | 提供路径选择器（调用系统文件夹选择对话框） | P0 |
| REQ-401c | 路径有效性实时校验（目录是否存在 + 可写） | P0 |
| REQ-401d | 自动同步开关（默认关闭） | P1 |
| REQ-401e | 配置持久化到应用数据库 | P0 |

### 3.2 导出功能

| ID | 需求 | 优先级 |
|---|---|---|
| REQ-402a | SummaryPanel 头部增加"导出到 Obsidian"按钮（图标按钮） | P0 |
| REQ-402b | 点击后执行导出流程：校验配置 → 创建目录 → 写入文件 | P0 |
| REQ-402c | 导出的 md 文件包含 YAML front matter | P1 |
| REQ-403a | 导出面板/对话框展示当前文献的所有聊天会话（带 checkbox） | P0 |
| REQ-403b | 支持全选/反选 | P1 |
| REQ-403c | 每个聊天会话导出为独立 md 文件，包含完整对话内容 | P0 |

### 3.3 自动同步

| ID | 需求 | 优先级 |
|---|---|---|
| REQ-404a | 监听 `finishStream` 事件，自动触发导出 | P1 |
| REQ-404b | 自动同步仅覆盖（不追加），以保持文件一致性 | P1 |
| REQ-404c | 自动同步失败时不弹窗打断用户，仅 console 警告 | P1 |

### 3.4 文件结构

```
{Obsidian Vault}/
└── 文献笔记/
    ├── {文献标题A}/
    │   ├── 总结.md           ← 默认总结
    │   ├── 总结-详细.md       ← 未来扩展
    │   ├── 总结-简略.md       ← 未来扩展
    │   ├── 对话-2026-03-20-10-30.md   ← 聊天记录
    │   └── 对话-2026-03-20-14-15.md   ← 另一次聊天
    └── {文献标题B}/
        └── 总结.md

## 4. 非功能需求

| ID | 需求 | 说明 |
|---|---|---|
| NFR-401 | 文件名安全 | 文献标题需 sanitize（去除 `/\:*?"<>|`，截断至 80 字符） |
| NFR-402 | 路径兼容 | 支持中文路径、空格路径 |
| NFR-403 | 原子写入 | 写入使用 write → rename 模式，避免部分写入 |
| NFR-404 | 无侵入性 | Obsidian 功能完全可选，未配置时不影响任何现有功能 |
```
